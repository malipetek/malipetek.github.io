How this blog is served from a Google Drive folder?

The Context

I want to have my blog in a Google Drive folder, so when I want to write something out, it's synced and immediately published. I built my github page from scratch and later redirected it to a firebase hosted page due to SEO problems. github pages SPA SEO problem.

The Problem

Google Drive API is a bit complicated. Sample codes and documentation I am sure very good but lacking clarity for very simple usage.

Method we are going to use it rather like having a personal API key for your Drive account. I would expect to find

How to

Here are steps to implement a client side folder access mechanism for your own Google Drive folder.

Step 1:

Go to Google Cloud Console, select a project you have or create one.

Step 2:

Click on the navigation menu and go to APIs & Services section.

Step 3:

Click on ENABLE APIS AND SERVICES button at the middle top.

Step 4:

When seacrh bar appear type "drive" and select Google Drive API from the results.

And finally click Enable.

Step 5:

Get back to API's section of your project.

Click on credentials from the left drawer.

If this is a firebase project like mine, you will see 2 keys already generated automatically.

Lets create a dedicated one anyways. Click on create credentials button on top of page: Click on API Key and close modal. Now you will see a new key in the list. Now click on pencil icon to edit your new key.

Step 6:

As we will be exposing this key to client side, it is wise to restrict this key to domain you will use it on.

Here I will restrict mine to mali-petek.web.app.

Also you might want to restrict your key to google drive API because anyone who navigated to your domain can send ajax requests from your domain as well.
